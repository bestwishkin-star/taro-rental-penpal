# Find Page Reading Feed Redesign

## 背景

当前发现页在完成“屋檐故事”社区化改造后，功能已经具备搜索、筛选、区域、排序和内容流，但视觉仍偏工具列表。页面缺少清晰的标题层级，筛选控件长期暴露导致第一屏拥挤，卡片字号和样式不一致，用户反馈为“UI 太丑、字体字号不对”。

本次目标是只重设计发现页 UI 和前端交互呈现，不改变后端接口、数据查询逻辑、路由结构或租房内容模型。

## 已确认方向

采用用户在视觉伴侣中确认的方案：

- 整体方向：B，笔记社区气质。
- 卡片密度：B3，单列阅读型。
- 顶部筛选：方案 2，筛选收进按钮。

页面应从“找房筛选工具”转为“屋檐故事阅读流”：第一屏更干净，内容更好读，搜索保留，复杂筛选默认收起。

## 体验原则

- 内容优先：首屏尽快露出第一条屋檐故事，筛选区不长期占据大面积空间。
- 字号稳定：移动端正文和摘要避免过小，建立固定字号阶梯。
- 阅读感强：卡片用单列大卡，标题、区域费用、摘要、互动信息分层展示。
- 社区气质：文案使用“一方屋檐下 / 屋檐故事 / 居住冷暖”语系，避免回到“房源列表”的表达。
- 触控友好：搜索、筛选按钮、筛选项和卡片点击区域满足移动端触控尺寸。

## 页面结构

发现页从上到下改为：

1. 顶部标题区：
   - 主标题：`一方屋檐下`
   - 副文案：`翻看城市里的居住冷暖`
   - 右侧或同层放置 `筛选` 按钮。

2. 搜索区：
   - 保留搜索输入。
   - placeholder 聚焦屋檐故事语境，例如 `搜索城市、地标、通勤、隔音...`。
   - 搜索框视觉更轻，避免和筛选按钮重复抢焦点。

3. 筛选面板：
   - 默认收起。
   - 点击 `筛选` 后展开一个轻量面板或下拉区域。
   - 面板内包含居住方式、费用范围、区域、排序。
   - 用户选择后立即更新现有 state，并可收起面板。
   - 保留清空区域或重置筛选能力。

4. 内容流：
   - 单列 `RentalCard`。
   - 卡片之间留出稳定间距。
   - 第一条内容在常见手机首屏内露出主体信息。

5. 状态反馈：
   - 加载态、空态、到底提示统一为轻量文字和更柔和的色彩。

## 字体系统

发现页内部采用一套局部字号阶梯，避免当前组件各自定义导致层级混乱：

- 页面标题：24px，700/800，行高约 1.25。
- 副文案：13px，400，行高约 1.5。
- 卡片标题：18px，700/800，行高约 1.35。
- 卡片元信息：13px，400/500，行高约 1.45。
- 卡片摘要：14px，400，行高约 1.6。
- 标签和辅助信息：12px，500，行高约 1.4。
- 搜索输入：14px，400。

所有字号不随视口宽度动态缩放，不使用负字距。

## 卡片设计

`RentalCard` 改为单列阅读卡：

- 卡片容器：
  - 宽度填满内容区。
  - 圆角控制在 14-16px，避免过度圆润。
  - 背景白或暖白，边框比阴影更轻。

- 封面：
  - 如果有图片，顶部展示固定比例封面。
  - 如果没有图片，用柔和色块占位，不制造空白或布局跳动。
  - 图片区域预留稳定高度，避免加载时内容跳动。

- 正文：
  - 标题最多自然换行两行。
  - 元信息一行优先展示 `城市/区域 · 费用 · 居住阶段`。
  - 摘要展示 `contentPreview`，没有时可退回 title，但实现时应避免重复感过强。
  - 底部展示讨论数、收藏数或凭证状态；凭证状态用文字徽标，不只靠颜色。

## 筛选交互

现有 `FilterChips`、`RegionFilter`、`SortBar` 不再全部常驻页面顶部，而是作为筛选面板内容。

建议实现为页面内展开面板，而不是新的路由：

- `showFilters` 控制展开。
- 面板放在搜索区下方。
- 面板展开时不阻断页面滚动，不需要复杂动画。
- 筛选项保持已有 state 和 `fetchRentals` 查询参数映射。
- `筛选` 按钮可显示当前是否有筛选条件，例如激活态或小圆点。

## 组件边界

主要修改：

- `apps/frontend/src/pages/find/index.tsx`
  - 增加标题区和筛选展开状态。
  - 调整控件排列。
  - 保持现有分页、刷新、搜索 debounce 和查询逻辑。

- `apps/frontend/src/pages/find/index.scss`
  - 定义发现页整体间距、标题区、筛选面板、列表间距和状态文案。

- `SearchBar`
  - 降低视觉重量。
  - 移除或弱化没有实际交互的筛选图标入口，避免和新 `筛选` 按钮重复。

- `FilterChips`
  - 作为筛选面板内的分组控件。
  - 修正当前 TSX className 与 SCSS 命名不一致的问题。

- `SortBar`
  - 作为筛选面板内排序控件。
  - 修正当前 TSX className 与 SCSS 命名不一致的问题。

- `RegionFilter`
  - 作为筛选面板内区域控件。
  - 保留清空能力。

- `RentalCard`
  - 重构为单列阅读卡。
  - 修正当前 TSX className 与 SCSS 命名不一致的问题。

## 数据流

数据流不变：

1. 用户输入搜索词，继续使用 debounce 更新 `debouncedKeyword`。
2. 用户调整筛选、区域或排序。
3. `loadPage(1, false)` 使用现有 `ListRentalsQuery` 请求数据。
4. 下拉刷新和触底加载保持现有行为。

本次不新增接口，不修改后端契约。

## 错误与空态

- 接口失败仍使用 toast，但文案保持屋檐故事语境。
- 空态文案保持 `暂无符合条件的屋檐故事`，提供重置筛选入口。
- 加载文案改为轻量状态，例如 `正在翻找屋檐故事...`。
- 到底文案避免生硬，例如 `这些屋檐故事先看到这里`。

## 测试与验收

实现完成后需要验证：

- `pnpm --filter @rental-penpal/frontend typecheck` 通过。
- 发现页可正常搜索、筛选、排序、选择区域、重置筛选。
- 下拉刷新和触底加载不回归。
- 有图和无图卡片都不发生明显布局跳动。
- 卡片标题、摘要、费用和筛选文字在常见手机宽度下不溢出。
- 页面中不再出现明显字号混乱：标题、元信息、摘要、辅助信息层级清晰。

## 非目标

- 不改后端接口。
- 不改数据库结构。
- 不新增测试文件。
- 不重设计详情页、首页或个人中心。
- 不引入新的 UI 库或图标库。
